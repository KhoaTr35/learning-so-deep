Model Backbone

This page summarizes the three model configurations used in the text track: a fully fine-tuned BERT classifier, a BiLSTM built on frozen BERT embeddings, and a lightweight learned ensemble that combines the strengths of both branches.

Architecture Overview

Each model plays a different role in the experiment: a strong fine-tuned Transformer, a lighter sequential baseline, and a small meta-learner for logit fusion.

Advanced

BERT Classifier

A standard fine-tuning setup built on the full bert-base-uncased backbone, optimized end-to-end for 20-way text classification.

  • Backbone: 12-layer Transformer encoder, hidden size 768, 12 attention heads.
  • Loading: AutoModelForSequenceClassification.from_pretrained().
  • Classifier head: pooled [CLS] output → Dropout(0.3) → Linear(768→20).
  • Tokenization: WordPiece with max length 256 and [CLS], [SEP], [PAD].
  • Training policy: all 109,497,620 parameters are updated during fine-tuning.
  • Mixed precision: AMP FP16 enabled with torch.amp.GradScaler.
  • Memory control: gradient accumulation of 2 steps keeps the effective batch size at 32.
109,497,620 trainable AMP FP16 Best accuracy: 70.23%
Baseline

BiLSTM + Frozen BERT Embeddings

A hybrid baseline that reuses BERT’s high-quality token embeddings but replaces the Transformer stack with a trainable recurrent backbone and attention pooling.

  • Embedding stage: frozen bert.embeddings module with 768-dimensional output.
  • Frozen parameters: 23,837,184 from word, position, token-type embeddings, and LayerNorm.
  • Recurrent encoder: 2-layer bidirectional nn.LSTM with hidden size 384 per direction.
  • Sequence handling: pack_padded_sequence respects the attention mask and valid lengths.
  • Pooling: learned scalar attention over timesteps followed by softmax-weighted summation.
  • Classifier: LayerNorm → Linear(768→384) → GELU → Dropout(0.3) → Linear(384→20).
  • Precision choice: trained in FP32 for numerical stability.
7,395,477 trainable 23,837,184 frozen Best accuracy: 68.00%
AMP is intentionally disabled here because BERT embedding LayerNorm uses eps = 1e-12, which is unsafe in FP16 and can lead to NaN gradients.
Ensemble

LearnedEnsemble (Meta-MLP)

A small fusion network trained on cached logits from the two backbone models. It does not fine-tune either backbone and only learns how to combine their outputs.

  • Inputs: frozen logits from the BERT classifier and the BiLSTM branch.
  • Meta representation: concatenated post-softmax logits with shape (B, 40).
  • Meta-MLP: Linear(40→80) → GELU → Dropout(0.15) → Linear(80→40) → GELU → Dropout(0.10) → Linear(40→20).
  • Training strategy: backbone predictions are pre-computed once and cached on CPU.
  • Compute cost: ensemble training is extremely light because only the small MLP is updated.
  • Checkpoint size: under 100 KB due to the tiny number of learned parameters.
7,340 trainable CPU-cached logits Best accuracy: 69.48%

Design Rationale

The three architectures were chosen to compare different ways of exploiting pre-trained language features under realistic hardware constraints.

Why these architectures?

  • Fine-tuned BERT serves as the strongest reference model because it directly adapts contextual Transformer features to the downstream task.
  • BiLSTM with frozen BERT embeddings tests whether strong token-level representations alone are enough when paired with a lighter sequential encoder.
  • The ensemble meta-MLP checks whether the attention-based view of BERT and the sequential bias of BiLSTM provide complementary decision signals.

Key engineering decisions

  • AMP is used only for BERT because the full model remains stable in mixed precision and benefits from reduced memory usage.
  • BiLSTM stays in FP32 to avoid underflow issues introduced by the frozen BERT embedding LayerNorm configuration.
  • Gradient accumulation allows BERT fine-tuning under a limited VRAM budget while keeping an effective batch size of 32.
  • Cached-logit ensemble training removes unnecessary GPU overhead and makes the fusion stage extremely efficient.
