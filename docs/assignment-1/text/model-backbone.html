Model Backbone

Architecture design for three models — a deep Transformer encoder, a bidirectional GRU with attention, and a learned ensemble meta-learner.

Assignment 1 / Text / Model Backbone

Architecture Overview

Advanced

TransformerClassifier

  • Embedding: nn.Embedding(45115, 256) with pad masking
  • Positional encoding: Sinusoidal (pre-computed, no learned params)
  • Encoder stack: 6 × TransformerEncoderLayer, Pre-LN, GELU activation
  • d_model: 256  |  Heads: 8  |  d_ff: 1024
  • Pooling: CLS token repr + masked mean-pool → concat (512-dim)
  • Classifier: LayerNorm → Linear(512→256) → GELU → Dropout(0.3) → Linear(256→20)
  • Dropout: 0.3 (embedding + encoder layers)
  • Final norm: extra nn.LayerNorm after encoder stack
16,426,004 parameters
Baseline

BiGRU (RNNClassifier)

  • Embedding: nn.Embedding(45115, 192) with pad masking
  • Recurrent layer: 2-layer bidirectional nn.GRU
  • hidden_size: 192  |  Directions: 2 → 384-dim output
  • Pack/pad: pack_padded_sequence for variable-length inputs
  • Attention: learned scalar attention over GRU output tokens
  • Pooling: softmax-weighted sum of all timestep representations
  • Classifier: LayerNorm → Linear(384→192) → GELU → Dropout(0.3) → Linear(192→20)
  • Dropout: 0.3 (embedding + between GRU layers)
9,851,541 parameters
Ensemble

LearnedEnsemble

  • Backbones: frozen TransformerClassifier + frozen BiGRU (no gradient flows into them)
  • Meta-input: concatenated logits from both models → shape (B, 40)
  • Meta-MLP: Linear(40→80) → GELU → Dropout(0.15) → Linear(80→40) → GELU → Dropout(0.10) → Linear(40→20)
  • Trainable params: 7,340 (meta-MLP only) — checkpoint is < 100 KB
  • Total params: 26,284,885 (including frozen backbones)
7,340 trainable (meta only)

Why These Architectures?

  • Transformer provides the global context capture expected from the "advanced" setting; CLS + mean pooling gives a richer representation than CLS alone.
  • BiGRU is chosen as baseline over simple LSTM because bidirectionality gives better context from both directions, and attention pooling avoids the GRU's vanishing-gradient pooling problem.
  • Ensemble tests whether complementary error patterns in the two backbones can be exploited with a tiny trainable head, avoiding expensive full re-training.

Key Design Decisions

  • Pre-LN encoder layers (norm_first=True) stabilise Transformer training without an aggressive learning-rate schedule.
  • pack_padded_sequence ensures the GRU never attends to padding positions, improving gradient quality for variable-length inputs.
  • Frozen backbones in meta-stage prevent catastrophic forgetting of individual model strengths during ensemble fine-tuning.
  • GELU activations used consistently across all classifiers for smoother gradient flow than ReLU.
