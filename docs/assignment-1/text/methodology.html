End-to-End Pipeline

1

Data Loading & Caching

20 Newsgroups fetched from Hugging Face (SetFit/20_newsgroups) and cached as local CSV. Falls back to sklearn if the hub is unreachable.

2

Text Preprocessing

Lowercase → strip email/news headers (regex) → remove URLs and email addresses → keep [a-z0-9'\s] only → normalise whitespace.

3

Vocabulary Construction

Count all tokens in training split; keep those appearing ≥ 2 times → 45,115 tokens. Add <PAD> (idx 0), <UNK> (idx 1), <CLS> (idx 2).

4

Sequence Encoding

Prepend <CLS> token, tokenise on whitespace, truncate / pad to 192 tokens. Both Transformer and RNN consume the same encoded sequences.

5

DataLoaders

Batch size 64 (GPU) / 32 (CPU). 2 persistent workers, pin_memory enabled on CUDA. Fixed generator seed (42) for shuffled training batches.

6

Training — individual models

AdamW + cosine-decay LR with linear warmup. CrossEntropyLoss with optional label smoothing. Gradient clipping at 1.0. Best checkpoint saved on test accuracy.

7

Ensemble Meta-learning

Load best individual checkpoints, freeze both backbones, train a small MLP meta-learner on concatenated logits. Only 7,340 parameters are updated in this stage.

8

Evaluation & Logging

Accuracy, macro precision/recall/F1, per-class F1, confusion matrix, macro one-vs-rest ROC AUC. All runs tracked with Weights & Biases.

Regularisation Strategy

  • Dropout (0.3): applied after embedding and inside attention/GRU layers to prevent co-adaptation.
  • Weight decay (1e-2): L2 penalty via AdamW keeps weight magnitudes in check.
  • Label smoothing (0.1 for Transformer): prevents overconfidence on noisy newsgroup headers.
  • Gradient clip (max_norm=1.0): stabilises training, especially for the deep Transformer stack.
  • Early stopping: saves the best checkpoint and halts if no improvement for 3–4 epochs.

Reproducibility Controls

  • Global seed 42 applied to random, numpy, and torch.
  • DataLoader uses a fixed torch.Generator with worker_init_fn to seed each worker.
  • GPU sanity check before training; falls back to CPU if CUDA is unstable.
  • AMP disabled by default for P100 compatibility; model checkpoints save best weights only.
  • All runs tracked in Weights & Biases with config snapshot for full reproducibility.
